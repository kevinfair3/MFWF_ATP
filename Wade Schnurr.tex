\subsection{Introduction}
In the Fall 2025 semester, during the initial research, the first attempts began to develop a proof-of-concept machine learning model using NACA airfoil data to predict four aerodynamic properties associated with each airfoil: coefficient of lift, coefficient of drag, pitching moment coefficient, and profile drag coefficient. The NACA airfoil dataset was used because they provide a large, structured, and well-documented aerodynamic database that was useful for early model development. The initial Long Short-Term Memory (LSTM) model underperformed compared to the current Multi-Layer Perceptron (MLP) model for reasons such as the non-sequential nature of the airfoil data. For the LSTM model to function, features were added to convert the non-sequential data into sequential data, which added unnecessary complexity to the model. The LSTM model was initially chosen as an exploratory model because LSTMs are known for their ability to predict nonlinear relationships. The current MLP neural network produces relatively high levels of accuracy when trained on the many data points contained in the NACA airfoil dataset.

In the Winter 2026 semester, the first objective was to finalize the conceptual validation of the MLP neural network using NACA airfoil data and to display the data in an intuitive way. The second objective was to alter the current model and/or develop a new nonlinear black-box aerodynamic model that could use dimensions and measurements from the ATP (force and wing tracking) and RRTS (moments) to predict the nonlinear three-dimensional aerodynamic forces and moments produced by the MFWF. The objective was to validate and compare the predicted forces and moments with the experimental data obtained from the ATP and RRTS. However, because the ATP and RRTS were still in development and were not producing meaningful experimental results at this transition point, the focus shifted to replicating a paper that used machine learning to predict dynamic aerodynamic forces and moments. The idea was that the replicated model could later be modified to accept future experimental data. The paper that was selected used a Probabilistic Recurrent-State Space Model(PR-SSM) to predict three-dimensional nonlinear wing forces and moments from an experimental dataset that also involved various wing trajectories.

A brief overview of neural-network terminology is useful for understanding the work presented in the following sections. In a simple feedforward neural network with backpropagation (as seen in Figure~\ref{fig:FFNN with BP}) the input information flows through a series of input, hidden, and output layers. The input data are received by an input layer, then passed through the hidden layers where the network learns patterns and nonlinear relationships, and then passed to the output layer where the prediction is produced. A layer is made up of multiple neurons that apply learned weights and biases to the input and then pass the result through an activation function, which is normally the same for all neurons in a given layer. Weights are numerical values used to scale how strongly the input affects the output to the next layer, while biases are numerical values used to independently shift the result. Activation functions are mathematical functions applied after the layer's linear transformation, allowing the network to model nonlinear relationships between inputs and outputs. During training, the model is updated through a process called backpropagation, which is used to update the network parameters based on the measured error after each prediction is made. An epoch is one complete pass of the training dataset through the network. These are some of the relevant concepts of a neural network. The aerodynamic quantities being predicted depend on nonlinear interactions between many variables such as geometry, operation conditions, and fluid-dynamic properties making neural networks an acceptable category of models for exploring aerodynamic predictions.

\subsection{MLP Neural Network - Methodology}
The initial machine-learning goal in Fall 2025 was to establish a proof of concept. The objective of this proof was to support the idea that a black-box aerodynamic model could be applied to predict aerodynamic forces from a simpler but still useful dataset. NACA airfoil data were chosen for the initial proof of concept because of the large, easily accessible, and well-documented dataset of standard steady aerodynamic quantities. Developing the MLP model as an intermediate step before data from, or similar to, the ATP dynamic data was useful because it allowed for development and understanding of machine learning processes. Starting with a simpler and well-defined dataset made it easier to understand, at a basic level, how the machine-learning process worked and how the model arrived at the predicted results from its inputs.

As previously stated, the dataset used came from Airfoil Tools \cite{airfoiltools}. All of the airfoil information that was exported was summarized into a Microsoft Excel file. The dataset included NACA airfoil identification (ID), angle of attack ($\alpha$), lift coefficient ($C_L$), drag coefficient ($C_D$), profile drag coefficient ($C_{DP}$), pitching moment coefficient ($C_M$), transition point on the upper surface ($Top_{Xtr}$), transition point on the lower surface ($Bot_{Xtr}$), and Reynolds number ($Re$). For the initial testing and development, a smaller sample of the airfoils was collected to form a dataset, which was later expanded for more intensive testing of the model. The final dataset used for training consisted of NACA 4-digit, 5-digit, 6-series, modified 6-series / 6A-series, H-series, and a few modified NACA variants, totaling 124 airfoils. One limitation of the final dataset used for training the MLP model was this size as it could have been larger. Of the 1638 airfoils available on Airfoil Tools, including non-NACA airfoils, only 124 were included in the final dataset \cite{airfoiltools}. The dataset covered Reynolds numbers on intervals ranging from 50{,}000 to 5{,}000{,}000, which exposed the model to a wide set of aerodynamic operating conditions. The selected inputs were airfoil ID, angle of attack, upper transition location, lower transition location, and Reynolds number, while the outputs were coefficient of lift, coefficient of drag, pitching-moment coefficient, and profile-drag coefficients. Before training could occur, the downloaded airfoil data needed to be preprocessed so that they could be interpreted by the machine-learning model. For each airfoil and angle of attack, the same airfoil ID and Reynolds number had to be applied to retain the relationship to geometry and operating condition.

The first type of model that was explored and tested was an LSTM architecture. The LSTM was considered because of in-class discussion and, in particular, its ability to predict complex nonlinear relationships. However, after the NACA airfoil dataset was examined further, it appeared that each sample could be referenced on its own and that the ordering of the rows did not represent a meaningful sequence or time history. LSTM models are designed to use sequential data, and therefore applying an LSTM to this dataset required an artificial interpretation of the data, which added unnecessary complexity. The next model that was applied was an MLP, since it uses independent tabular samples. The MLP uses a feedforward architecture in which the selected inputs can be taken directly from the dataset without the use of an additional memory structure like the LSTM required. Therefore, for the proof-of-concept study, the MLP model was eventually selected because it better matched the type and structure of the dataset.

The final version of the MLP architecture was a feedforward neural network with three hidden layers. The hidden layers used 128, 128, and 64 neurons respectively, which then led to the output layer containing four neurons, each corresponding to one predicted aerodynamic coefficient. The total input space used four numerical features, Reynolds number, angle of attack, upper transition location, and lower transition location, in addition to the one-hot encoding used to represent the airfoil ID. The activation function used in the hidden layers was the Rectified Linear Unit (ReLU), while the output layer used a linear activation function to directly predict continuous aerodynamic coefficients. Before training, scaling was applied to the numeric input features and the target coefficient values using a MinMaxScaler transformation to scale the values to the zero-to-one range. The dataset was split into training (70\%), validation (15\%), and test (15\%) subsets using a reproducible random split with a fixed random seed. Training was performed for a maximum of 300 epochs with a batch size of 64. Validation performance was continuously monitored, early stopping was implemented with a patience of 20 epochs and restoration of the best weights, and a ReduceLROnPlateau callback was used to lower the learning rate after 8 epochs without validation-loss improvement. The loss function used was a weighted mean-squared error, with increased weights assigned to $C_D$ and $C_{DP}$ because they were, on average, predicted less accurately than the other outputs.

\subsection{MLP Neural Network - Results}
One of the main ways the performance of the MLP model was evaluated was through the use of predicted versus true plots, or parity plots, for each output coefficient \cite{WadeProgressPresentation3_2025,WadeProgressPresentation4_2025}. The plots use the true coefficient values on the horizontal axis and the model predictions on the vertical axis. A point that lies on the 1:1 line represents a perfect prediction. The parity plots were useful in rapid analysis of the model and in presentations because they gave a quick visual indication of prediction quality, spread, and any departure from the ideal predictions. In addition, training-history curves, or loss curves, were used to assess the stability and efficiency of MLP learning. The training and validation loss curves were examined to determine whether the model converged smoothly, whether the validation performance closely followed the training performance, and whether there was evidence of overfitting or unstable optimization.

A more detailed residual analysis across the input variables was not completed, which means that any concentration of error with respect to specific airfoil characteristics or operating conditions could not yet be identified with a high degree of confidence.

The MLP results were compared against the earlier LSTM trials to support the selection of the updated model. The comparison summarized in Table~\ref{tab:mlp_lstm_comparison} shows that the LSTM configuration without airfoil ID produced the weakest predictive accuracy, while including airfoil identity in the dataset increased the averaged overall $R^2$ by about 31.4\%. The change from the LSTM with airfoil ID to the MLP was much smaller, with the averaged overall $R^2$ increasing by about 1.2\%. Even with the smaller numerical gain, the MLP achieved the strongest averaged predictive performance while also using a simpler architecture that more naturally matched the non-sequential dataset \cite{WadeProgressPresentation3_2025,WadeProgressPresentation4_2025,MLPMetricsJSON_2026}. The results obtained in the Fall semester established a technical baseline for later aerodynamic modeling work with the subgroup. At the same time, the Fall work clarified the limits of what could be learned from static NACA airfoil data alone. Although the MLP model performed well on the steady airfoil dataset, strong performance on static airfoil data did not resolve the final ATP prediction objective because the dataset did not capture the unsteady, three-dimensional aerodynamic behavior of the MFWF system. The current MLP model should therefore be interpreted as a useful proof-of-concept surrogate model rather than a direct solution to the final ATP prediction problem. An additional outcome of the MLP work was the development of a Streamlit-based interface that displayed aerodynamic coefficient predictions in an interactive format and improved the interpretability of the results across the input space.

\subsection{PR-SSM Paper Replication - Methodology}
The paper ``State-space aerodynamic model reveals high force control authority and predictability in flapping flight'' \cite{PRSSMFlapping2021} was selected as the paper to replicate because it was closer to the ATP objective than the NACA airfoil study. The Probabilistic Recurrent State-Space Model (PR-SSM) and dataset used in the paper shared many similarities to the types of data the MFWF test stands were anticipated to produce. The Fall MLP addressed steady tabular airfoil data, whereas the MFWF requires dynamic prediction of flapping-wing forces and moments. The PR-SSM study also treated the aerodynamic prediction as a sequence-modeling problem, which more closely aligned with flapping-wing motion than a static regression model. The paper therefore provided a stronger connection to the MFWF application because it predicted three-dimensional force and moment coefficients varying with time rather than steady two-dimensional coefficients.

The PR-SSM study predicted three-dimensional time-varying aerodynamic forces and moments from flapping-wing trajectory data. In the outputs, the targets were reported as $CF_x$ (x-direction force coefficient), $CF_y$ (y-direction force coefficient), $CM_x$ (x-axis moment coefficient), $CM_y$ (y-axis moment coefficient), and $CM_z$ (z-axis moment coefficient). The force and moment coefficients are dimensionless, normalized forms of aerodynamic forces and moments. They are predicted instead of dimensional values because they provide a more general representation of the aerodynamic behavior and make comparison across different operating conditions and configurations easier. The flapping-wing trajectory dataset supplied with the PR-SSM repository \cite{BayizPRSSMRepo} included both the model inputs and the corresponding measured output quantities used for training and evaluation. The inputs were organized as sequences and represented wing-motion history and state information. The local copy of the repository also contained the MATLAB input files as well as output folders containing training logs, saved configurations, checkpoints, loss histories, and postprocessed metrics. An example of a representative strong training configuration, was iteration 35. Interation 35 used a latent-state dimension of 12, one recognition layer, 100 inducing points, 30 samples, and seed 123 (fixed randomized seed initialization between iterations). The main limitation of the current dataset is that it still came from the replicated PR-SSM study rather than from synchronized ATP and RRTS measurements.

The replicated PR-SSM used the same core state-space structure as the original codebase. The representative strong training configuration used nine filters, kernel size six, pool size five, and stride one. The activation function choice was important because the model had to capture nonlinear aerodynamic behavior while still keeping stable sequence learning throughout the trajectories. Tanh was used in the representative configuration. The PR-SSM code handled most data preparation internally, so the local setup focused on trying to maintain consistency between the input trajectories, hyperparameters, and postprocessing procedures. The quality of the training was evaluated through saved loss histories and postprocessed prediction metrics, with $R^2$ and RMSE from the the held-out test trajectories being the most useful measures. Early stopping was active in the representative run and used the validation split with patience of 3 epochs and a minimum improvement threshold. A stepwise decaying learning rate was also used during training. During the search to optimize the hyperparameters, the dataset partitions and seed initializations were fixed across runs so that the hyperparameter configuration changes could be compared evenly. The robustness to other splits and seed initializations still needs to be tested once a few hyperparameter configurations have been optimized.

The latest work replicated the original PR-SSM framework directly in several important respects. The local project retained the provided repository structure, the supplied MATLAB input files, and the TensorFlow 1.14 implementation used by the original codebase. Other elements had to be adapted and developed for local use. Some of the changes included the file structure, experimental run tracking and comparison processes, hyperparameter-search processes, and the summary of tracking across many iterations of output folders. The postprocessing and visualization stage was also expanded through new local scripts, which were used to generate plots, compare test runs, and summarize metrics more efficiently than the original workflow alone. An important improvement made over the semester was transitioning from informal exploratory testing to a more controlled comparison. The comparison spreadsheets became more structured over time, which brought to light issues and fixes such as the need for fixed dataset partitions and fixed seed initializations during hyperparameter search. Those changes were needed because the objective extended beyond rerunning the paper configuration and included identifying promising settings and building a process that could later transition to ATP- and RRTS-relevant data. The PR-SSM methodology was therefore closer to addressing the overall objective because it modeled nonlinear, time-varying outputs from sequential motion data rather than isolated (non-sequential) steady-state points.

Several limitations remain. A main weak point in the representative PR-SSM run was the uneven prediction quality across the five outputs. In particular, $CF_y$ and $CM_x$ were predicted with lower accuracy than the other three outputs. The fact that the dataset was experimental could also play a part in the variation between the predicted and measured results. Slight variations between tests could also contribute to the imperfect predictions. By splitting the dataset into three fixed subsets of training, validation, and testing trajectories during the hyperparameter search, the comparison across runs was made more controlled. However, a final optimized configuration still needs to be tested across multiple random seed initializations and alternative random trajectory partitions. The local changes made to the codebase were thought to be reasonable and necessary, for example to reduce training time, although they also made it more challenging to achieve an equivalent or superior reproduction of the results from the paper. A question that remains is how the model will generalize beyond the provided flapping-wing dataset. Despite those limitations, the results from the current development indicate that the viability of a PR-SSM-style sequential model is encouraging. The model addresses the objective using a dynamic form and already performs fairly well on a related flapping-wing dataset (the dataset used in the paper). A synchronized ATP and RRTS dataset containing trustworthy kinematics and the corresponding force and moment measurements in a machine-learning-ready format still remains missing. Some possible avenues for future validation include trajectory-level generalization, sensitivity to alternative dataset partitions, and direct testing on ATP (three-dimensional force and wing trajectory) and RRTS (three dimensional moment measurements) measurement data.

\subsection{Conclusions and Recommendations}
Across the two semesters, the main contribution of the neural-network subgroup was the start of the development of a clearer model-selection pathway for aerodynamic prediction. The Fall MLP work created a baseline workflow and demonstrated that steady airfoil data were sufficient for building a preliminary proof-of-concept model. The Winter PR-SSM work improved the physical relevance of the model and data by moving from steady, static coefficient prediction to unsteady, dynamic force and moment coefficient prediction on flapping wing trajectory data.

The final metrics imply two useful outcomes. The Fall MLP confirmed that the basic machine learning process was viable. The PR-SSM achieved average test set $R^2$ near 0.95 on a dynamic flapping wing dataset, demonstrating that the model learned meaningful aerodynamic patterns. Based on the results, the current work can reasonably claim to have established a credible machine learning based modeling path and to have demonstrated that PR-SSM-style dynamic modeling can perform strongly on a related flapping-wing dataset. However, at this time the current work cannot confidently claim validated prediction accuracy on ATP (3D force measurements and wing  trajectory data) and RRTS (3D moment measurements) generated data or full robustness across different dataset partitions, seeds, and operating conditions.

The recommended next step is to continue the PR-SSM hyperparameter search under controlled conditions, such as using fixed datasets and seeds until a clearly superior configuration can be identified. Validation through held out test trajectory performance, residual analysis, and comparison of the outputs should continue and be improved, with targeted effort focused on the weaker performance in $CF_y$ and $CM_x$. Another important improvement is to prioritize using experimental data that are more representative rather than only increasing the complexity of the model. For example the current flapper only uses one trajectory and the model contains many trajectories. When an optimized set of hyperparameters has been identified under controlled testing, the next step should be to rerun the model across multiple random seed initializations and trajectory partitions. Reporting of uncertainty should also be improved by comparing performance across multiple seeds, trajectory partitions, and repeated runs rather than depending on a single best-case result. A high-priority recommendation is to create a synchronized ATP force measurement, ATP wing tracking, and RRTS moment dataset in an appropriate machine learning ready format, or as a start utilize the anticipated file type and data format. The quality of the aerodynamic outputs should be checked before very large training runs, and the dataset should use consistent naming, synchronized measurements, sufficiently broad operating-condition coverage, and clearly defined dataset partitions. it is also recommended to log records of tests, for example by updating the comparison spreadsheets.
